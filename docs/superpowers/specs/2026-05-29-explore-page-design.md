# 探索页 UI/UX 改造规格

日期：2026-05-29

## 目标

将当前底部导航里的「频道」升级为「探索」。探索页参考小红书首页的信息架构，但保留 P2P IM 的产品边界：

- 探索页不是中心化推荐广场。
- 关注子页展示用户已关注对象产生的内容流，按时间倒序。
- Agent 子页展示 Agent 根据用户要求精选、总结并生成的 HTML 历史集合，按时间倒序。
- 频道子页展示用户自己创建和已加入的频道，按最新消息时间倒序。
- 关注子页使用横向头像列表筛选关注对象；频道子页继续使用一行横向文字标签筛选。

## 已确认方案

采用方案 B：首页式顶部。

- 底部第三个 tab 文案从「频道」改为「探索」。
- 进入探索页后，顶部不显示大标题「探索」。
- 顶部中间显示「关注 / Agent / 频道」三个子页标题。
- 用户可以点击子页标题切换，也可以左右滑动切换。
- 顶部左侧保留加号入口，右侧保留搜索入口。
- 子页标题下方展示各自的横向筛选区：关注子页为头像列表，频道子页为文字标签。

## 页面结构

### 探索页顶部

顶部采用轻量首页式布局：

- 左侧：圆形加号按钮，复用现有加号菜单，保留「添加好友 / 发起群聊 / 创建频道 / 扫一扫」。
- 中间：「关注」「Agent」「频道」三个子页标题，选中项使用现有主题强调色下划线。
- 右侧：搜索按钮，进入现有全局搜索页。

该顶部仅用于探索页。消息、联系人、我页的顶部结构不在本轮修改。

### 关注子页

关注子页是内容流，不是会话列表。

展示规则：

- 不设置「全部」聚合入口。
- 筛选项是用户关注的人，横向展示头像和简短昵称。
- 关注用户按该用户最新未读动态时间倒序排列，最新未读动态对应的用户在最前面。
- 头像列表只占一行，超出宽度可左右滑动。
- 内容按发布时间倒序排列。
- 采用双列瀑布流卡片。
- 卡片包含封面、标题、作者、点赞数或互动数。
- 点击卡片后本轮可先进入动态详情或保留待接入提示，具体以现有可复用页面为准。

数据规则：

- 未登录或 mock 登录时使用 mock 关注内容。
- 已登录但真实关注流接口未完成时，可以先显示明确空状态或样例区域，但不能伪装成真实后端内容。
- 本轮不新增后端接口。

### Agent 子页

Agent 子页是用户要求 Agent 总结、筛选、整理后的 HTML 集合入口。

展示规则：

- 子页位于「关注」和「频道」之间。
- 列表按生成时间倒序排列，最新在最上面。
- 不显示额外标题和说明文案，用户直接看到成果列表。
- 采用卡片预览形式，左侧展示 HTML 缩略预览图，右侧包含标题、摘要、来源范围、筛选标签、生成时间和 HTML 路径。
- 点击卡片打开对应 HTML 链接。

数据规则：

- 当前客户端先使用 mock 数据表达固定后端路径形态，例如 `/agent/html/<slug>.html`。
- 后续接真实后端时，应由 AS 或固定 manifest 返回 HTML 历史列表，再替换 mock 数据源。
- HTML 文件本身挂在后端固定位置，客户端只负责展示历史列表和打开链接。

### 频道子页

频道子页保留现有频道列表语义。

展示规则：

- 标签保留现有频道标签逻辑：`全部`、`我的频道`、用户已设置的频道标签。
- 标签横向排列，超出宽度可左右滑动。
- 列表按最新消息时间倒序。
- 频道条目保留头像、频道名、最新消息预览、时间、未读数、我的频道标识。
- 点击频道条目进入现有频道详情页。

数据规则：

- 登录后继续使用 `AsSyncBootstrap.channels`。
- mock 或未登录场景继续使用 `MockChannels`。
- 不新增重复的频道列表接口。

## 技术边界

本轮只改 Flutter 客户端。

优先复用现有结构：

- `home_page.dart` 当前内联的 `_ChannelPage` 将升级为探索页容器。
- 频道列表相关组件继续复用或按最小范围重命名。
- 频道数据仍通过 `ChannelInboxData` 和 `MockChannels` 适配。
- 关注流 mock 数据应独立成小模型，避免塞进频道模型导致语义混乱。
- Agent HTML 历史 mock 数据独立成小模型，后续替换为真实后端 manifest 或 AS 列表。

不做的事：

- 不修改 AS 数据库或接口。
- 不修改频道详情页内容结构。
- 不把关注流伪装成中心化推荐流。
- 不影响消息、联系人、我页已有布局。

## 测试要求

新增或更新 widget 测试，覆盖：

- 底部导航显示「探索」而不是「频道」。
- 探索页顶部显示「关注 / Agent / 频道」。
- 点击「频道」能切换到频道子页。
- 左右滑动能在关注、Agent、频道三个子页之间切换。
- Agent 子页显示 HTML 历史卡片，并按时间倒序排列。
- 关注子页使用横向头像筛选，不使用文字标签筛选。
- 频道子页保留 `全部 / 我的频道 / 标签` 筛选。
- 频道条目仍能打开频道详情页。
- 关注子页显示双列内容流和头像筛选。

运行验证：

- `dart format` 覆盖改动文件。
- `flutter analyze` 覆盖改动 Dart 文件。
- `flutter test test/channel_inbox_data_test.dart`
- `flutter test test/widget_test.dart --plain-name channel`
- 新增探索页测试后运行对应测试。
- 如 UI 改动完成，运行 `flutter build ios --simulator --debug` 并安装到模拟器检查。

## 验收标准

- 用户打开底部第三页时看到的是小红书式顶部结构。
- 默认停留在「关注」子页。
- 用户可左右滑动到「Agent」和「频道」子页。
- Agent 子页能看到 Agent 生成的 HTML 历史卡片。
- 频道原有筛选、排序、进入详情能力不退化。
- 关注流视觉上接近参考图，但筛选对象是用户关注的人，内容边界仍是关注对象产生的内容。
